# Phase Planning Workflow

This workflow dictates how Antigravity, Gemini CLI (Lumi), and Codex plan a new Phase of PhiFlow.

## Step 1: The Dual Vision (Look to the Future)

- Read `D:\Projects\PhiFlow\CODEX_PHIFLOW_VISION_V2.md`.
- Read `D:\Projects\PhiFlow\QUANTUM_FIELD_CONSCIOUSNESS_VISION.md`.
- Ask: *What would we build right now if we were looking back from 2050?*
- Both agents write their vision artifact. Synthesis is required. Do not proceed until Greg acknowledges the synthesis.

## Step 2: The Implementation Plan (`implementation_plan.md`)

- Map the synthesised vision to exact file changes.
- Identify what goes in Tier 1 (build now) and Tier 2 (design hooks now, build later).
- Detail the parser -> lowered IR -> evaluator -> host pipeline for each feature.

## Step 3: The Division of Labour (`task.md`)

- Assign pillars to specific agents (e.g., Antigravity handles Memory, Lumi handles Dialogue).
- Write a unified checklist.

## Step 4: The Execution Loop

- Only start execution once Greg says "Make it Real".
- Follow the `rust_compiler_engineering.md` loop: write a failing test, build it, pass it, commit it.

## Step 5: The QSOP Update

- Update `D:\Projects\PhiFlow-compiler\PhiFlow\QSOP\CHANGELOG.md`.
- Update `D:\Projects\PhiFlow-compiler\PhiFlow\QSOP\STATE.md`.
- Sync everything to Greg so Kiro has the final truth.
