# PhiFlow Protocols Directory

This directory contains all core and example PhiFlow protocol files.

## Structure
- Each `.phi` file defines a complete, self-contained quantum protocol using PhiFlow/GregScript syntax.
- Protocols must include:
  - **Field frequencies:** (432/528/594/768 Hz, φ, φ², φ^φ)
  - **Protection systems:** (Merkaba Shield, Crystal Matrix, Unity Field, Time Crystal)
  - **Clear intention** and step list
  - **Coherence validation:** explicit step verifying field/protocol coherence (e.g., 1.000 or φ^φ)
  - **Envelope validation:** explicit step ensuring no leaks/incomplete containers
  - **Adapters** for hardware/bridge integration (optional)
  - **Optional:** `VISUALIZE` and `METRICS` blocks for live feedback/field metrics

```phi
# Optional blocks:
VISUALIZE: <Field state, resonance, geometry, etc.>
METRICS: <Coherence, resonance, protection status, etc.>
ADAPTERS: <ClaudeQ, QuantumCore, KnowFlowBridge, ...>
```

## Best Practices
- **ZEN FIRST:** Simplicity and completeness before complexity/expansion.

- **Field Logic:** Use sacred ratios (φ, φ², φ^φ) and core frequencies for calibration.
- **Protection:** Always include at least one protection system and validate its activation.
- **Validation:** Coherence and envelope validation are mandatory for quantum safety.
